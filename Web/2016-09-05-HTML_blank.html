HTML 的Hacker Friendly target 的 "_blank"

這個問題主要是HTML 裡面我們常用的 


<a target='_blank'>

這個Tag 有點小問題

因為當我們 Click 了一個含有 target="_blank" 的一個 tag 後，Browser 就會去調動 HTML 本身 API 的 window.opener ( 詳情請見 w3m )
接著讓Broser Call function 就會去開一個新的分頁


這時惡意的攻擊者就可以將Payload 寫在新打開的網站中，之後window.opener 這段API 就會強迫本來打開這段Code 的原始網頁，reload 成一個新打開的URL 位置

以下為使用範例 - 影片Demo

a.html 


<html>
<head></head>
<body<
    This is my test page 
    <a target="_blank" href="test.html">Click here</a>
</body>
</html>

test.html 


<html>
<head>
    <script language="javascript">
        window.opener.location = 'https://www.google.com'
    </script>
</head>
<body>
    This is my second test page
</body>
</html>
