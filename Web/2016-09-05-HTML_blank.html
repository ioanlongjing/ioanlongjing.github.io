HTML 的 Hacker Friendly target 的 "_blank"

這個問題主要是 HTML 裡面我們常用的 


<a target='_blank'>

就這個 Tag 有點小問題 ＠＠

因為當我們 Click 了一個含有 target="_blank" 的一個 tag 後，Browser 就會去調動 HTML 本身 API 的 window.opener ( 詳情請見 w3m )
接著 Browser Call function 後就會去建立一個新的分頁 ( New Tab )

這時如果要連入的網站，是惡意的攻擊者建立的，就可以將 Payload 寫在新打開的網站中
在開啟有 Payload 的 Web 後，window.opener 就會強迫原始網頁 reload 成黑客設定好 URL 位置，至於這有什麼害處呢？

目前想到就是可以偽裝成釣魚頁面或是用 CSRF 拿到人家的Cookie、Session或是其他的token，如果還有新的或有趣的奇淫技巧，歡迎一起來討論囉。
