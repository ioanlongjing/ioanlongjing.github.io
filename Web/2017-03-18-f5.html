[ Web ] Struts 2 防禦 ( 用 F5 ASM 解決)
Web, Struts 2, F5

不知道大家有沒有 follow 到前一段時間流行的 S2-045 Struts 2 的漏洞，拿了多少站 XD

這次原本想說來記錄跟筆記一下 S2-045 原理
不過由於太多人聊過了, 那... 我們改一下，來講防禦好了

首先... 我們先有一台有這漏洞的 Server


嗯，很明顯，我們可以執行任意 Commamd

假設今天漏洞忽然噴出來了，而我們由於各種歷史原因，不太適合直接改 Code 或是需要原本 Coding 的人權限才能修改
那... 那樣很麻煩啊，等到找到人，或是可以改 Code 的公文下來，駭客都拿來當成雲端伺服器了
於是我們這次改用 F5 的 ASM 來嘗試緊急擋下來

假設今天漏洞忽然噴出來了，而我們由於各種歷史原因，不太適合直接改 Code 或是需要原本 Coding 的人權限才能修改
於是我們這次改用 F5 的 ASM 來嘗試先擋下來

當我們 login 後點到 Security -> Options -> Application Security -> Attack Signatures -> Attack Signature List


並在 Rule 上寫 ： pcre: "/[oO]gnl/";
( 因為 Ognl 表達式通常不會在 user request 時來，因為 server 在build 網頁時就會被 compiler 掉，如果 user 有用到，可能在嘗試對 server code injection ，所以我們在 user 傳到 server 時，WAF 上就把它擋掉，避免傳到 server )


再來我們加一個 Group ，如果有其他知道的東西要擋，也可以一併加入
選項位置在 Security -> Options -> Application Security -> Attack Signatures -> Attack Signature Sets


之後將 Policy 的規則都設好了，我們將阻擋開啟，位置在 Security -> Application Security -> Pollcy Building -> Learning and Blocking Setting


當規則都弄好了，只要決定你的 policy 要放在哪個 Virtual Server 把它 enable 就好了


同樣的 payload 我們在試一次，你看，攻擊被擋掉惹

        {
            log local0. "Found Struts S2-045 attack! Rejecting a request with Content-type [HTTP::header "Content-Type"] to  [HTTP::uri]  from  [IP::client_addr]"
