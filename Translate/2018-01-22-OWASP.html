OWASP TOP 10 A3 - Sensitive Data Exposure

原文來自 : OWASP (Github)


Even anonymous attackers typically don’t break crypto directly. They break something else, such as steal keys, do man-in-the-middle attacks, or steal clear text data off the server, while in transit, or from the user’s client, e.g. browser.
Manual attack is generally required.

一般來說匿名的攻擊者不會沒辦法直接破解加密。他們會從例如說瀏覽器，做如偷密鑰、中間人攻擊、或是從 Server 偷取明文的資料之類的事情。

手動攻擊是很常見的。

Over the last few years, this has been the most common impactful attack. The most common flaw is simply not encrypting sensitive data. When crypto is employed, weak key generation and management, and weak algorithm usage is common, particularly weak password hashing techniques. For data in transit server side weaknesses are mainly easy to detect, but hard for data in rest. Both with very varying exploitability.

在過去這幾年，這是一很普遍的攻擊方式。最常見的就是沒加密敏感資料。使用密碼時，加密過程中被生成跟管理的是使用比較弱的 key 與比較弱的加密演算法。對於資料傳輸伺服器的弱點比較容易被檢測到，但對於資料比較放心。但其實兩者有很大的不同。
